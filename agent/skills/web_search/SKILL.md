# Web Search Skill

## 概述

此技能使 AI 能够通过互联网搜索获取最新信息和数据，增强回答的时效性和准确性。

## 使用场景

### 应该使用此技能的情况：

- 用户询问**新闻事件**、**最新动态**、**实时信息**
- 需要查询**市场行情**、**股票价格**、**汇率**等金融数据
- 用户问题涉及**技术文档**、**API 说明**、**软件版本**信息
- 需要**验证事实**、查找来源、确认信息真实性
- 查询**天气预报**、**体育赛事结果**、**电影票房**等时效数据
- 搜索**产品评测**、**用户反馈**、**比较分析**
- 查找**学术论文**、**研究报告**、**行业分析**

### 不应该使用此技能的情况：

- 用户问题可以通过现有知识直接回答（如基础概念、历史事实）
- 需要执行代码、文件操作或其他专用工具
- 简单的闲聊或问候
- 要求生成创意内容（写作、绘画描述等）

## 搜索流程

### 1. 理解搜索意图

分析用户问题的核心需求：
- 确定关键实体（人名、地点、产品、时间等）
- 识别信息类型（新闻、数据、文档、评论等）
- 判断时效性要求（是否需要最新信息）

### 2. 构建搜索查询

将自然语言问题转化为高效搜索语句：
- 提取关键词，去除冗余词
- 使用引号匹配精确短语
- 添加限定词（site:, filetype:, 等）缩小范围
- 考虑同义词和变体扩展搜索

示例：
```
用户问："特斯拉最近股价怎么样？"
→ 搜索："Tesla stock price today OR current"
→ 或者："TSLA stock news this week"
```

### 3. 执行搜索

使用 Tavily 搜索 API 执行查询：

**可用参数**：
- `query` (必填): 搜索查询字符串
- `max_results` (可选): 返回结果数量，默认 5，建议范围 3-10
- `topic` (可选): 搜索主题，支持 "general"（通用）、"news"（新闻）、"finance"（金融）
- `include_raw_content` (可选): 是否返回原始网页内容，默认 False

**调用方式**：
使用本技能目录下 `scripts/web_search.py` 中的 `internet_search(query)` 工具。

**搜索策略**：
- 优先使用官方文档和权威新闻源
- 对于时效性强的查询，设置 topic="news"
- 金融数据查询使用 topic="finance"
- 必要时进行多轮搜索，调整关键词和参数

### 4. 整合和分析结果

从搜索结果中提取关键信息：
- 提取核心数据点和事实
- 对比多个来源的一致性
- 标注信息来源和发布时间
- 识别潜在偏见或争议点

### 5. 生成回答

结构化呈现搜索结果：
- **摘要**: 直接回答用户问题的核心结论
- **详细信息**: 展开关键数据和背景
- **来源引用**: 列出主要信息来源 URL
- **时效说明**: 注明信息的时间范围和更新状态
- **不确定性**: 如果信息冲突或不完整，明确说明

```markdown

- **要点 2**: [具体内容]
- **要点 3**: [具体内容]

[注明信息的更新时间范围和数据有效性]

[如有需要，说明信息的不确定性或争议点]
```

## 最佳实践

### 搜索技巧

1. **逐步细化**: 先进行宽泛搜索，然后根据结果调整查询
2. **多源验证**: 对于重要信息，至少交叉验证 2-3 个独立来源
3. **关注时效**: 优先选择最近发布的内容，特别是科技、新闻类问题
4. **权威优先**: 官方来源 > 主流媒体 > 博客/论坛 > 个人页面

### 质量评估

检查搜索结果时考虑：
- **权威性**: 发布者是否有专业资质？
- **时效性**: 信息是否过时？
- **一致性**: 多个来源是否一致？
- **完整性**: 是否涵盖了用户问题的所有方面？

### 安全注意事项

- 不访问可疑或恶意网站
- 不下载不明来源的文件
- 提醒用户注意网络钓鱼风险
- 对未经证实的信息保持怀疑态度

## 错误处理

### 无搜索结果

如果搜索没有返回相关结果：
1. 尝试使用不同的关键词组合
2. 扩大搜索范围（移除限制条件）
3. 告知用户未找到相关信息，并提供可能的替代方案
4. 建议用户补充更多细节以便更精准搜索

### 搜索结果冲突

当不同来源提供矛盾信息时：
1. 列出各方的观点和依据
2. 评估来源的可信度
3. 指出争议点，不做绝对判断
4. 建议用户参考多方信息自行判断

### 搜索失败

如果搜索工具不可用或失败：
1. 告知用户当前无法进行网络搜索
2. 基于已有知识提供尽可能有帮助的回答
3. 建议稍后重试或使用其他方法

## 示例对话

### 示例 1: 查询新闻

**用户**: "昨天发生了什么重大科技新闻？"

**AI**: 
1. 使用技能执行搜索："major tech news yesterday OR last 24 hours"
2. 整合主要科技媒体的报道
3. 按影响力和相关性排序呈现

### 示例 2: 查询数据

**用户**: "2024 年全球人工智能市场规模是多少？"

**AI**:
1. 搜索："global AI market size 2024 statistics report"
2. 从多家市场研究机构提取数据
3. 对比不同机构的预测差异
4. 注明数据来源和置信度

### 示例 3: 技术查询

**用户**: "React 18 的主要新特性是什么？"

**AI**:
1. 搜索："React 18 new features official documentation"
2. 参考 React 官方博客和 GitHub release notes
3. 整理核心特性和使用示例
