# GraphRAG Knowledge Query Skill

## 概述

此技能为Agent提供**GraphRAG企业知识图谱本地检索能力**，通过调用专用Python脚本读取图谱索引文件（parquet），结合社区等级、响应格式参数，从已构建的企业知识库中精准获取信息，为用户提供权威、无幻觉的知识问答服务。

## 使用场景

### 应该使用此技能的情况：
- 用户询问**公司制度、请假流程、报销规范**等管理知识
- 查询**技术文档、项目规范、安全制度**
- 咨询**培训资料、操作手册、管理办法**
- 需要基于**GraphRAG+Neo4j结构化知识图谱**的专业问答

### 不应该使用此技能的情况：
- 用户问题与知识库无关（闲聊、天气、编程、娱乐等）
- 需要实时新闻、网络动态数据
- 无需图谱检索的基础常识问答
- 需要执行文件操作、网络搜索、代码运行等其他技能
- 创意写作、文案生成类非知识查询需求

## 图谱查询流程

### 1. 解析用户查询
分析用户问题核心需求：
- 提取企业知识核心关键词（制度、流程、规范等）
- 识别查询类型：单次问答 / 批量查询
- 确定回答精细度需求

### 2. 构建查询参数
将自然语言问题转化为脚本标准入参：
- 必传：用户原始查询问题
- 可选：社区响应等级（1-5）、回答格式类型
- 批量查询：传入问题列表

示例：
用户问："请假流程是什么"→ 标准参数：query="请假流程是什么", community_level=2→ 响应格式：Multiple Paragraphs

### 3. 执行知识图谱查询
调用`scripts/graphrag_query.py`脚本执行检索，底层联动GraphRAG本地搜索API与Neo4j图谱数据：

**核心工具函数**：
1. `graphrag_knowledge_search`：单次知识图谱查询

**可用参数**：
#### 单次查询参数
- `query` (**必填**): 用户知识库相关问题字符串
- `community_level` (**可选**): 社区响应精细度，1-5，数值越大回答越具体，默认=2
- `response_type` (**可选**): 回答格式，支持：
  - `Multiple Paragraphs`：多段落（详细，默认）
  - `Single Paragraph`：单段落
  - `Single Sentence`：单句子

**调用逻辑**：
脚本自动加载GraphRAG配置文件 → 读取图谱索引数据 → 执行本地搜索 → 返回结构化结果

### 4. 结果处理与整合
- 解析脚本返回的结构化字典（success/answer/error）
- 提取有效回答，过滤错误信息
- 批量查询自动汇总所有结果，统计成功数量
- 按指定格式生成用户友好的回答

### 5. 生成最终回答
基于检索结果结构化输出，保证专业、简洁、准确。

## 输出格式

### 单次查询标准响应

```markdown
## 知识图谱查询结果

### 核心发现
[GraphRAG 提炼的精简答案，直接回应用户知识库相关问题]

### 详细信息
- **要点 1**: [制度、流程、规范相关细分知识点]
- **要点 2**: [操作要求、注意事项、补充说明]

### 数据来源
1. GraphRAG+Neo4j 企业知识图谱(本地索引) - 基于结构化知识库检索生成

### 时效性说明
本地图谱静态知识库，信息长期有效，无实时时效限制

### 备注
内容由知识图谱检索生成，具体操作请以公司最新制度为准
```

## 最佳实践

### 查询技巧
精准提问：使用企业知识关键词，提升检索准确率
等级适配：简单问题用等级 2，复杂流程用等级 4-5
格式选择：快速咨询用单句 / 单段落，详细学习用多段落
批量使用：一次性提交多个相关问题，提升效率

### 质量评估
权威性：答案完全来自本地构建的企业知识图谱
相关性：严格匹配用户企业知识相关问题
完整性：覆盖核心知识点与注意事项
无幻觉：不生成图谱以外的虚假信息

### 约束规范
仅处理企业知识库领域专业知识查询无匹配数据时不编造答案不调用任何外部网络资源，纯本地检索

## 错误处理

### 配置文件缺失
错误场景：settings.yaml 不存在处理方式：返回错误提示，引导初始化 GraphRAG 项目

### 索引文件缺失
错误场景：output 目录 /parquet 文件不存在处理方式：提示用户先执行 graphrag index 构建索引

### 查询执行失败
错误场景：脚本运行异常、API 调用失败处理方式：返回具体错误信息，不返回无效回答

### 参数异常
错误场景：community_level 超出 1-5 范围处理方式：自动重置为默认值 2，继续执行查询
